Step by step process to submit a Country Contract in MERC+

  1. If your PC is setup for ENGLISH (UNITED STATES) you may proceed to step 2. If other than ENGLISH (UNITED STATES) please proceed to step 1.1
    1. Click START button
    2. Select CONTROL PANEL
    3. Select REGIONAL AND LANGUAGE OPTIONS
    4. Select ENGLISH (UNITED STATEs)

       
    5. Click OK

       
     
  2. Prepare the contract in Microsoft Excel using the format which can be downloaded here
     
  3. After reading, delete the information in cells A3, A4 and A5.
         The downloaded format should be used as it has all columns pre-formatted for ideal data transmission.
    All columns are formatted as TEXT with the exception of column G which is formatted as a number with 2 decimals.
    < Formatting columns as text will allow leading zeros when needed, as in mode '03' and code '0950' >




     
  4. You are now ready to add the contract line items using the following rules:
    • Header line must be exactly as the sample - no changes in spelling etc. except putting your email address in cell I1
    • Cells for email address on contract lines should be left blank (see sample below)
    • File cannot include multiple excel sheets
    • MANUAL = MAER unless notified otherwise.
    • CONTRACTAMOUNT column cannot contain commas - must be formatted as numeric currency.
      Examples -  23 must be written as 23.00   and   14.2 must be written as 14.20
    • EFFDATE and EXPDATE must be formatted as a total of 8 characters,  YYYYMMDD without hyphens or slashes
    • EFFDATE must be prior to EXPDATE
    • EFFDATE must be equal to or later than contract upload date
    • a Blank in CONTRACTAMOUNT is not allowed
    • a Zero in CONTRACTAMOUNT will be updated in database as .00 and will ensure shop cannot enter any material cost for that code.

      Resulting excel contract should look like the sample below
               (note the leading zeros are in place and email address is only listed on the header line)



       

  5. When contract is complete save by clicking 'FILE' at top left of screen and select 'SAVE AS'
     
    6.
  6. In the 'SAVE AS' window give the file a name with up to 8 characters with no spaces.
     
  7. Click the 'Save as Type' dropdown and select default    (Microsoft Excel Workbook)(*.xls)
    (NOTE - We save the file twice, first as an .xls file in case errors are found in .csv file.
      This allows for editing and re-creation of the .csv file without starting over.)
     
  8. Repeat step 8  and 10, this time selecting  CSV (Comma delimited)(*.csv)
    Note - you many have to scroll down in order to see .csv

     
  9. Click 'SAVE'
  10. When you get the following 'popup' click - OK



     
  11. Which will bring up a 2nd 'popup' - click YES



     
  12. When you close the file - it may ask a 2nd time if you want to save - say no.
     
  13. Check the format by right clicking the .csv file & selecting 'OPEN WITH' and select NOTEPAD from the menu.
    NOTE - file will open with Excel but leading zero's will NOT appear - that is why you MUST check with NOTEPAD not Excel.

     
  14. The file should look like this:
    SHOPCODE,EFFDATE,EXPDATE,MANUAL,MODE,REPAIRCODE,CONTRACTAMOUNT,ed@hotmail.com
    209,20041221,20060131,MAER,03,0940,23.12,
    209,20041221,20060131,MAER,03,1111,42.00,
    209,20041221,20060131,MAER,03,1112,43.10,
    209,20041221,20060131,MAER,00,0940,16.00,
    209,20041221,20060131,MAER,00,1112,23.00,
    209,20041221,20060131,MAER,00,1113,23.00,
     

    These are obvious errors that should be corrected before sending in the .csv file

    SAMPLES CAUSE
    209,20041221,20060131,MAER,03,1111,42.00, Good - no errors
    209,20041221,20060131,MAER,3,1111,42.00, Leading zero missing from mode
    209,20041221,20060131,MAER,03,1111,42, CONTRACTAMOUNT is 42 should be 42.00
    209,20041221,20060131,MAER,03,1111,42,00, CONTRACTAMOUNT contains a comma not a period
    209,2004/12/21,20063101,MAER,03,1111,42.00, Dates are not in strict YYYYMMDD format
    209,20041221,20060131,MAER,03,1111,42.00, US,20041221,20060131,MAER,03,1111,42.00,001,
    209;20041221;20060131;MAER;03;1111;42.00; Fields separated by other than a comma (see step 1)

     Other obvious errors:
        - Missing email address from header
        - Adding email address to other lines

    Do not attempt to correct .csv file - instead make corrections to .xls and re-create .csv file.

     

  15. When checking is complete - send the contract, in the .csv format , via e-mail to your area EMR manager.
     
  16. If REGIONAL AND LANGUAGE OPTIONS were changed in step 1 - you may reverse the process.
