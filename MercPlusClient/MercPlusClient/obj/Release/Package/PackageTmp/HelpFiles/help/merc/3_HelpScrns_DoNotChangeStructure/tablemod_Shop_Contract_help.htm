OVERVIEW

This screen will be used to manage contracts between Vendor and local MSL (a country). It provides users with an interface where users can Query, Update, Delete and Add new Contracts.

AUTHORITY 

Only Admin, CPH, Area, Country, Region, MSL (Location/Local EMR) will have access to the Shop Contract Screens.

QUERY 

For querying Shop contract Screen 'Shop’ is Mandatory (Cannot do a search without selecting a Shop) - Also search can be further filtered by Repair Code and/or mode.

The result set will be sorted in the following Order:

  • 1 Shop

  • 2 Manual

  • 3 Repair Code

  • 4 Mode

  • 5 Eff Date

The result set should display the following:

  •  Shop

  •  Repair Code

  •  Amount (Shop Currency Amount)

  •  Manual

  •  Mode

  •  Effective Date

  •  Exp Date

  •  Link to edit

  •  Link to delete

Note: The Shop in the Drop Down will be all existing Shops (Also based on User Permissions) and Modes in the Moe Drop Down will be all existing Modes (Actives & inactive)

 

UPDATE 

Users will be able to view and or edit Shop Contracts by

  •  clicking on the edit link next to the record

    The only editable fields will be:

    •  Amount

    •  Expiration date

  •  Selecting the checkboxes and clicking on update selected

    The only editable field will be:

    • Expiration date

Note: If Exp date less than current date no edit link/check box will be available

Note: Expiration date must be greater than or equal to the effective date.

Note: Expiration date must not be less than Current Date

Note: Date format expected as YYYY-MM-DD

Note: Shop has to be updated as inactive to edit contracts

ADD NEW  

Users will be able to add new Shop Contracts by clicking on the ‘Add New Contract’ button provided on the Search Screen which will take them to an “Add New Page’.

No 2 contracts should fall in the same date range. There can be 2 contracts for the same:

  • Shop

  • Manual

  • Mode

  • Repair Code

But different date ranges and or one active and other expired

Note: Manual / Mode / Repair Code must be valid in Repair Code table

Note: Expiration date must be greater than or equal to the effective date.

Note: Expiration date must not be less than Current Date

Note: New contract effective date must follow last existing contract

Note: Date format expected as YYYY-MM-DD
