QUERY SCREEN   

  • The query screen is the heart of the process. It is where decisions as to what estimates are approved or rejected are made. Each local approver has access to the shops in his area. Besides the local approver, only the approver's superiors can see his shops.
  • Estimates can be filtered by selecting a SHOP, ESTIMATE ENTRY DATE, CUSTOMER, EQUIPMENT SIZE, EQUIPMENT TYPE, EQUIPMENT SUB-TYPE, EQUIPMENT NUMBER, VENDOR REF. NUMBER, MODE, COUNTRY CLUSTER, COUNTRY, LOCATION or any combination of same.
  • There are also 'canned' query types which can list all estimates of the selected status - again this can be filtered by selecting other fields as well.
  • The main query is the DAILY QUERY but other other queries can be constructed as well.
  • There is also a sort by drop down which can be selected to return the rows in that particular order. The list of sorting options available are :-
    • Amount Only
    • Equipment Number Only
    • Total Hours Only
    • Mode Only
    • Size/Type Only
    • Vendor Ref. No. Only
  • The status column contains status description for the row.
  • Approval of estimates. Submitted estimates are either (200) PENDING or (310) SUSPENDED
    • PENDING estimates can be handled in 2 ways
      1. Multiple PENDING estimates can be approved, but not rejected, from this screen by clicking the check box in the 1st column and by then pressing the APPROVE ALL CHECKED button.
      2. A PENDING estimate can be approved or rejected by clicking on the Location field, this causes the estimate to display.
    • SUSPENDED estimates cannot be managed from this screen. See #2 above.
      • Once the user is at the individual estimate screen, he can navigate within the selected group of estimates without having to return to the query screen.

    Daily query 

    The DAILY QUERY for local MSL is the mainstay of the day to day approval process. It gives the user the chance to call up with one put of a button (in most cases) all the estimates that are awaiting his decision.

    The user can "filter" the DAILY QUERY by selecting a single shop from the dropdown and or a mode prior to querying. This is to accommodate larger areas that handle multiple shops or for areas that have individuals only responsible for certain modes (ie. reefer repair). This filtering allows the user to only see the estimates he has to manage.

    For the vendor, the DAILY QUERY gives him a summary of what has been approved or rejected. It also lists the un-submitted estimates (drafts) and those which have been submitted but not approved.

     

     

    Query a specific equipment ID or reference number
     

        A specific estimate or work order can be queried by inputting the equipment ID or reference number in the appropriate field on the query screen. The search can be narrowed by selecting the shop number or location if desired.
