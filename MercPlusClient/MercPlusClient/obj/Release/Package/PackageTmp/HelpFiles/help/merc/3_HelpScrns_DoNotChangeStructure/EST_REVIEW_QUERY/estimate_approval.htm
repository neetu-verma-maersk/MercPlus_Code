The approval process

An estimate is called up by clicking on the currency link for that estimate on the QUERY SCREEN.

While looking at the estimate, the approver can make the following decisions:

  • Approve the estimate as is.
  • Reject the estimate with comments.
  • Suspend the estimate with comments (if it is presently in Pending status).
  • Leave the estimate as is - that is postpone the decision.
  • Change the 3rd party cause field and/or add a 3rd party location.

Please note - the only fields the approver can change are the 3rd party cause field and/or add a 3rd party location. Any other problems with the estimate cannot be changed by anyone except the vendor. An estimate with an error must be rejected to the vendor who can then change and re-submit. For this reason it is essential that the comments which must accompany any rejection should be clear and concise.

After the decision has been taken on an estimate, the approver can then do any of the following:

  • Move to the next estimate in the existing query by clicking the > button at the top or bottom of the estimate screen.
  • Move to the previous estimate in the existing query by clicking the < button at the top or bottom of the estimate screen.
  • Move to the last estimate in the existing query by clicking the >> button at the top or bottom of the estimate screen.
  • Move to the first estimate in the existing query by clicking the << button at the top or bottom of the estimate screen.
  • Clicking the [NEW QUERY] button at the top or bottom of the estimate page will return the user to the QUERY screen.

Approving Suspended estimates

  • SUSPENDED estimates can only be disposed of by reviewing them on screen. SUSPENDED estimates are estimates which require a closer scrutiny before approving - they are not necessarily incorrect.
  • Disposition of SUSPENDED estimates - SUSPENDED estimates can be:
    • Approved
    • Rejected
    • Passed to GLBEMR (comments will be required)

Passing estimates to GLBEMR or CENEQUSAL

  • Mandatory forwarding to GLBEMR or CENEQUSAL
    • Some estimates will arrive with status of (320) SUSPEND TO CPH these can be disposed of in 2 ways:
      1. Reject back to vendor with comments
      2. Forward to GLBEMR with comments or Forward to CENEQUSAL with comments [if allowed - else via GLBEMR]
    • These estimates will be decided by CPH
      • If approved - the estimate is routed direct to the vendor (local MSL can see if they query)
      • If rejected - the estmate comes back to local MSL, who can add his comments and then reject to vendor.
  • Voluntary forwarding to GLBEMR or CENEQUSAL
    • Estimates can be forwarded to GLBEMR or CENEQUSAL if the local MSL rep decides circumstances warrant. The comments which must be added should reflect why this decision was taken.

 

 

 

How to reject an estimate

  • An estimate can be rejected to the vendor by adding comments and clicking the REJECT button.
  • Clicking the REJECT button changes the estimate status to (100) REJECTED and makes it visible to the vendor. It will appear on his DAILY QUERY.
    • The vendor then has the option to (logically) delete the rejected estimate or correct and resubmit.
  • Once an estimate has been rejected - the status cannot be changed. If an estimate is rejected in error - you must contact the vendor and ask them to resubmit.

 

 

COMMENTS

  • EXTERNAL COMMENTS - comments that will be forwarded to the vendor. These comments stay with the estimate and can be viewed by all who can see the estimate.
    • When rejecting an estimate to vendor, MERC+ requires that comments be added prior to forwarding. These comments should inform the vendor why the estimate was rejected and suggestions as to what they can do to alter the estimate for re-submission.
  • INTERNAL COMMENTS - comments that will be visible only to MSL personnel. These comments stay with the estimate and can be viewed by all MSL personnel who can see the estimate. There may be more than one internal comment for an estimate.
    • When forwarding to GLBEMR or CENEQUSAL, MERC+ requires that comments be added to the estimate prior to forwarding. These comments should inform the vendor or CPH of your thoughts about the estimate.
