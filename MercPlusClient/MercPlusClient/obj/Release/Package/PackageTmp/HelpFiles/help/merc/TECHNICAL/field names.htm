INSPECTIONS Inspection information  pertaining to the Chassis equipment. The business needs to keep track of when inspections are performed on Chassis and charged as a repair on a WO.  Pulling reports showing the frequency of inspections being done may show abuses being done in terms of un-necessary inspections.  A query will also enable Shops to determine if an inspection actually does need to be done on the Chassis, or if the sticker on the equipment was missing or in error.  
LABOR RATE The hourly amount charged to a Customer, by a Shop for repair work done on equipment.  The Rate is determined by the type of equipment being repaired and the type of hours the mechanic worked. Each Shop may have different rates depending upon the Customer the work is being performed for, and the rates vary depending on the type of equipment being worked on as well as the kind of hours being worked.  The business also has the need to retain the rates for a given time period to enable them to increase the rates begining with a set date, and not have the new rate reflect on work done prior to the effective date.  
LOCATION A geographical area in the world. This will be used to group Shops in a particular area of the country as managers oversee all Shops in a Location.  
MANUAL A guideline to equipment repairs. A reference guide of repair codes.  Existing manuals in WMNR will be replicated down at start of system.  
MANUFACTURER The producer of the equipment. The table is also used to identify Genset’s manufacturer and warrenty period.  
MASTER PART Master Part is a list of part codes    
MESC1TS_CUST_SHOP_MODE This entity defines a relationship between CUSTOMER, SHOP and MODE.  It is used for Non STS Modes to identify non standard time billing that will need approvals from a different security profile set up specifically  for Non STS Approvers.  It is also used to define a PAYAGENT associated with each combination for proper payment of a workorder in RRIS system. This will  be used in the system logic  to identify the workorders a Non STS Approver will have the authority to approve, and to define the payagent associated to a given Customer, Shop and Mode on a workorder.  
MODE A grouping of similar repair requirements, either pertaining to a particular type of equipment or pertaining to a particular type of repair work. MODE_PREP_TIME Maximum time that can be allocated for a given type of preparation work being done on an equipment PrepTime defines the Prep Codes to be sent to RKRP when transmitting a WO.  The Prep_Time_Max is the sliding scale indicator, so if the Workorder is calculated as using 2 preparation hours, the process must look for the appropriate Prep_Cd that represents 2 hours.  For instance, the table would be looked up to find the record where 2 hours is less than or equal to the Prep_time_Max, ordered by Prep_TIme_Max to be able to retrieve the first one only.  This Prep_Cd would be added to the WO data when transmitted to RKRP.  
MODEL A list of Manufacturer’s Model numbers. This table is used to find the first digit of the repair code that must be used when repairing gensets whose equipment number falls within the given range.    
NON STS CODE This entity represents work that was done and is being invoiced that does not include repair work to a piece of equipment These codes will be set up to represent miscellaneous work done and being invoiced back to the Customer.  The work is not necessary done as a repair nor is it done under a stand time and material rate, it is done as a flexible rate.  
PARTS_GROUP A List of parts group    
PAYAGENT The Payagents defined in RRIS accounting system which determine the characteristics of a payment, such as how the payment is made and  the organization responsible for making the payment. RRIS requires different information in the data transmission depending on the payagent involved

A history of all comments and remarks applied against a WO will be retained for future reference.  This will include the system generated information regarding the reason a WO was suspended for manager’s review.  
REPAIR CODE EXCLUSIONARY A designated relationship between one repair codes and another, which indicates the codes that are excluded from being added to a workorder when a specific repair code is already used. Certain Repair Codes are created that include activities that could be listed separately as their own Repair Code.   When one of these codes is used on a WO, it must be checked that the the Exclusionary Code  does not appear on the WO as well.  
REPAIR MASTER PART An entity that links workorder Repair and master parts list    
REPAIR_CODE An identifier of work performed, including description, maximum quantities and amounts. These codes will be initially downloaded from the WMNR mainframe system.  They represent standard time and material charges that are not to be exceeded when the repair code is used on the WO.  The business will also have the ability to create their own specific Repair Codes that are not used in RKRP, and associate the correct RKRP repair code with it.
