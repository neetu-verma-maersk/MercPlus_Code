Estimate Management

  • Approve - to give the approval for repair according to estimate

  • Suspend - to suspend the estimate for further action. There may be many reasons for this, e.g. wait for the additional missing information like unit costs etc.

  • Update - there are two items available for updates by local Maersk office               
    1.  Cause -     meaning the cause of the damage. Please note that after make changes, you must click the button of 'update', or information won't be transferred.
    2.  3rd Party Port - if the 3rd party damage is confirmed, please fill in the location code of the  3rd party responsible and click the button of 'update', or information won't be transferred.

  • Reject - to reject an estimate because of certain criteria are not met.

  • FWD to EMR mng - when the estimate is beyond the approve limits of the local Maersk office, click the button to forward the estimate to EMR mng.

  • FWD to CENEQULOS - forward the estimate to CENEQULOS if it involves the container sales.

  • EDIT - Opens the estimate in edit mode.

  • <<< -  Moves to first page of the queried records
  • << - Moves back one page
  • New Query - system will go back to the Query screen for a new query.
  • >>    - Moves to the next page
  • >>>  - Moves to the last page of the query
  • Audit Trail – summarized when/what is every updates on the estimates and by which party this step if performed
  • Print - click the button to print the query result.
  • Equipment Number – container number.
  • Equipment Information – system automatically reveals the container profile after entering the Equipment Number. These information are retrieved from RKEM.
    • Type – container type, e.g. CONT/REEF
    • Profile – gives the information of the series the container belongs to
    • Size – container size, e.g. 40/9’6
    • Material – the material the container made of, e.g. steel
    • Box MFG –manufacturer of the container
    • In Service – when the container start the service
    • Reefer make/model – the manufacturer of the reefer container and the model information
    • PTI Date – the latest Pre Trip Inspection date, if available
    • Genset make/model 
    • For Sale - Whether container updated for sale or not, the field will state Y or N (yes or no)
    • Owner - This field informs you of owner type e.g. OWN, LT, ST, SUB, ONE, VSA, SO ( Maersk owned, Long term leased, Short term leased, )
    • On Global Hunt - Displays information whether container is updated for redelivery (Off Hire)
    • Fixed Cover - If container is under fixed cover, the amount will be stated here. Fixed cover is to be regarded as a insurance policy. This means that every container updated for redelivery and with fixed cover, can be returned as it is, and Maersk Line will only have to pay the amount updated under fixed cover, regardless whether container is damaged or not.
    • DPP - If container under DPP, amount will be stated here. DPP stands for Damage Protection Program which means that in case of redelivery, damages up to the amount mentioned under DPP is covered, and that Maersk Line only has to pay repair costs exceeding DPP amount.
    • At off Hire Location - Informs you if container can be redelivered (Off Hired) at your location.
    • Empty - If container empty, the field will display a Y (Yes)
    • For Refurb - Shows if container is to be refurbished.
    • Refurb Date - This field will display the last refurbishment date, if available.
  • Status - the estimates status, e.g. Rejected, Approved, Draft etc.

  • Shop Code – the only code recognized by the system, which stands for your repair shop.

  • Previous Status – work order status for the same equipment ,created before the current one (Previous Status is freezed after current work order approval).

  • Previous Date – work order change date for the same equipment ,created before the current one (Previous Date is freezed after current work order approval).

  • Previous Location – work order change date for the same equipment ,created before the current one (Previous Location is freezed after current work order approval).

  • Revision Number – Number of times, a work order status is changed.

  • Mode – type of the container, e.g. 43 Reefer unit, 03 Steel Box etc
  • Vendor Ref  – This field has space for 10 characters which can be filled in by the vendor, or it can be left blank. If the field is left blank, MERC+ will generate one on its own. The number will be used as reference in future payments.
  • Estimate NO - the sequence number given by the system on current estimate
  • Currency – system will give the default currency according to the respective repair contract.
  • Completion Date - the day when this estimate was completed.
  • Pcs – the number of the damaged parts under same repair code.
  • Damage Code – the type of damage to be repaired .
  • Repair Code – the code from the Repair Manual categorizing the type of the container damage.
  • Repair Loc Code – locates damages within an area as large as a complete face of the container or as small as a nominal 1200 mm x 1200 mm (4 ft x 4 ft) square, or even less for the main components located on its edges (rails, corner posts).
  • Man Hours per Piece – the number of the working hours needed for the repair activity on each piece. MERC+ will only accept up to the maximum hours per code, as stated in the STS manual.
  • Description – Text description of repairs performed
  • TPI Indicator – indicates the responsibility of the repair .
  • Material Cost per Piece – the expense of the material per each necessary under the respective repair cods
  • Total per Code – total repair expense under the repair code, including the material cost as well as the man hours.
  • Total Wear & Tear
  • Total Third Party
  • Total to be paid excluding tax/VAT
  • Total to be paid including tax/VAT
  • Total cost of repair excluding tax/VAT
  • Total cost of repair including tax/VAT
  • External Remarks – all information the repair shop wants to share with local Maersk office, including general conditions of the container, description of the main damage, suggestion of the repair, the revision made on the estimates as well as the status of the repair activities etc.
  • Internal Remarks - all information you what to share with the other internal parties, e.g. GLBEMR, CENEQUSAL
