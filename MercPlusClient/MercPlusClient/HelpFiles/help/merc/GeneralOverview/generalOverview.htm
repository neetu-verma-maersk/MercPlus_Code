OVERVIEW - MERC+ is a database which allows approval and storing of maintenance and repair estimates which have been input directly into the system by local vendors. This input takes place over the internet or via a interface to MERC+ from the vendor's proprietary system.

Data Entry by vendor/shop 
Estimate entry can be input by vendors in 3 ways     

  • Browser input - for smaller vendors. All they need in internet access and they can input estimates into MERC+.
    • Advantages - Cheap and little preparation is needed – once vendor has a shop number he can begin to input estimates.
    • Limitations - only as fast as the internet connection, cannot be used if internet or MERC+ is down.
    • Support – via E-mail.

     

  • Proprietary system - Vendor uses own system and transmits estimates via E-mail. Vendor can build in as many safeguards as desired into his own system. (The more he builds in, the fewer rejects he will experience.)

How input is transmitted to MERC

In addition to inputting directly via internet browser, there are 2 formats available in order for vendors to send estimates from their own system to MERC+;

  • E-mail attachment – estimates can be put into text file (.txt) format. How often - transmissions can be received at any time. It is suggested that they take place nightly. Longer intervals will make error correction more difficult.
  • Web Services - similar input/output to E-mail but more prompt response.

Acknowledgement of receipt - When MERC+ receives a text file – it sends a return message acknowledging receipt to the originator.

If something goes wrong - If vendor does not receive acknowledgement he can re-transmit to MERC+.

Validation

PRE-ACCEPTANCE - Before an estimate is accepted into MERC+, the equipment ID is validated by RKEM. In addition, all the modes, codes, pieces etc. are validated against business rules.

  • MERC+ tables are checked for:
    • correct labor rate
    • excess hours
    • excess pieces
    • excess shop materials
    • incorrect part numbers
    • in some cases - is part number acceptable for the mode/code selected.
  • RKEM is checked for
    • Correct unit that is owned/leased by MSL
    • Unit size, type
    • Lease status
    • Sale status
    • Box/machinery make/model.

  • PREP TIME - does not have to be inserted as it is calculated automatically. Details on PREP TIME

If not valid – invalid estimates are rejected by MERC & rejected to vendor for correction or deletion. This process happens overnight. Vendor is notified via E-mail with appropriate error message informing what is wrong with the rejected estimate(s). Rejected estimates is deleted from MERC+ thus any rejected estimate will have to be re-submitted.

Note – where possible MERC+ will run as many validations as possible – this to limit possibility of estimates being re-rejected multiple times.
Vendor has option of correcting error & re-submitting or deleting an estimate.
Once estimates are in MERC+, they pass through for approval or investigation and/or rejection by local Maersk Line agency. 

SAFEGUARDS

  • SAFEGUARDS in MERC+
    • MODES
      • LIMITED CHOICES DISPLAYED - Modes available for selection will be limited to the modes which might apply (standard repair modes plus off hire mode)
      • CAUTION - However in the end the user can still select the improper "correct" mode.
     
    • STS CODES
      • PIECES - The number of times a repair code can be used is limited to the maximum allowed per the STS.
     
    • HOURS
      • Number of 10th's of an hour which can be used is capped by the STS. However the 10th's per piece can be lowered.
      • Maximum hours per piece can be calculated by MERC+ by clicking a button on the estimate entry screen.
     
    • SPARE PARTS
      • SHOP SUPPLIED MATERIAL - this figure, reported on a per code basis, in local currency. It covers non-numbered parts and materials and is capped by:
        • Global ceiling for the mode/code which cannot be exceeded.
          • It is EXTREMELY IMPORTANT that MSL personnel understand the global ceiling is a "safety net" only. It is NOT the standard amount to be charged. Personnel should not mention the global ceiling to vendor - they should constantly stress that the actual cost should be entered.
        • Contract ceiling (if a contract is in place for the shop). Contract ceiling will overrule global ceiling.
      • NUMBERED PARTS
        • Maersk supplied - These are numbered parts owned and supplied by MSL. They may (in the case of reefer probes and chassis tires in NAM) be monitorial MSL supplied parts.
        • Shop supplied - These are numbered parts supplied by the vendor
        • There will be 3 types of shops - each type will handle numbered parts in a different manner.

 

 

 

FILTERING OF ESTIMATES

Once the estimate has passed all validations, it is accepted into MERC+ and is immediately available for viewing by local MSL. The local EMR representative will have a one click process to see a summary of all estimates in queue awaiting disposition. These estimates will be divided in 3 categories.

1. SUSPENDED - An estimate in SUSPEND status, while not necessarily incorrect, is an estimate which requires closer scrutiny than average to insure it is correct. An estimate could be in SUSPEND for a number of reasons. Reviewing the reasons for SUSPEND shows that the local EMR representative has a lot of flexibility in deciding what estimates he wants to take a closer look at. Some of the reasons to SUSPEND are:
  • 3rd Party Codes - Has an STS code which is potential 3rd party – manager will pass w/o for payment (work has already been done) and investigate, get TIR's etc.
  • VSA - Chassis in RKEM as VSA – manager will check if on hire to MSK & follow up.
  • ONE, SUB and SO - Containers which are in RKEM as ONE, SUB-leased or Shipper Owned.
  • Over shop repair limit – manager checks for approved estimate Manager flagged STS code – manager will follow up. (This level is set by manager by shop/mode)
  • Over shop supplied parts for w/o – manager will follow up, if needed, with vendor.
  • Done on non-straight time – If required, manager will check to see if overtime was approved. Note - this SUSPEND cause can be switched off at the shop level.
  • Over shop parts limit for individual STS code – manager will follow up, if needed, with vendor.
  • Manager Suspension - manager can put pending w/o into SUSPEND status to do more investigation. He must add internal comments to w/o as reminder of why he took action.
  • Global Hunt = Y
  • Flagged by vendor - An estimate which the vendor thinks needs close review (this estimate will include the vendor's remarks as to why he thinks the estimate needs review)
  • Marked FOR SALE
  • etc.???

    Handling SUSPENDS - Estimates which are SUSPENDED can be either:

  • Approved
  • Rejected
  • Passed on to GLBEMR for review
  • Passed to CENEQUSAL for possible sale (direct or via GLBEMR depending on whether this is allowed for the location)

2. SUSPENDED TO CPH - An estimate which is suspended to CPH is one which meets one or more of the following criteria:

  • Over the CPH limit for the piece of equipment
  • Over the special remarks limit for the piece of equipment
3. PENDING - An estimate which is PENDING is an "ordinary" estimate. It is ordinary in that it meets none of the SUSPEND criteria.
What is gained by looking at 40-50 identical estimates - each for pre-tripping a reefer for one hour? MERC+ segregates the potential problems from the ordinary estimates.

DISPOSITION OF ESTIMATES - Depending on the settings for his location, the local representative can either:

  • APPROVE - this routes the approved estimate to the vendor.
  • REJECT - this routes the rejected estimate, with comments added by MSL, to the vendor.
  • FORWARD TO GLBEMR - for them to make the decision
  • FORWARD DIRECT TO CENEQUSAL - [if permitted] for decision on sale of a unit.

If, for any reason, a vendor alters an estimate which had been approved, that estimate is automatically re-submitted as an unapproved estimate. It does indicate that the estimate has been altered.

Approval process

The approval process will take place on a daily basis. Timely approval will allow managers to better plan their day and accomplish more. Once the day’s approvals have been settled, manager can be occupied elsewhere. This in contrast to poring over a week’s worth of work at one sitting.

Managers will NOT be able to correct any error. This eliminates manual cutting of invoices leading to confusion when invoice is short paid.

In short, estimates will either be approved or rejected.

HOW WE CONTROL APPROVALS

Each vendor/shop has following controls:

  • Shop Repair Ceiling 

     

    • What is it ? - Submitted estimates that exceed this ceiling are accepted in SUSPEND status. Generally manager should give suspended estimates extra scrutiny. Specifically estimates exceeding estimate ceiling should have an approved estimate for work that has been done.

     

    • What it does - Flag's estimates that have been estimated or should have been. Ceiling enables EMR manager to better manage his shops by setting and re-setting the 'trust level' of these shops.

     

    • Who sets it ? - EMR manager sets appropriate ceiling for each shop he is responsible for. He can adjust these levels to react to changing situations.

     

  • STS code shop parts ceiling 

    • What is it ? - A maximum amount which can be set for each STS code which has associated parts. If shop supplied parts for a code exceed its ceiling, w/o is accepted into MERC in SUSPEND status. A ceiling on every code is not necessarily desirable.

    • What it does - tells how much a vendor can charge without flagging the manager. It is used in conjunction with  PCT Material Factor 

    • Who sets it ? - Corporate will set appropriate parts ceiling for codes it selects. Ceilings can be adjusted (by code) to fit changing circumstances

    Total Material Limit

    • What is it ? - Maximum amount for total shop supplied parts for w/o. If amount is exceeded, w/o is accepted into MERC in SUSPEND status for review by manager.
    • What it does - Flags w/o's with potential excessive parts cost for manager's review.
    • Who sets it ? - EMR manager sets an appropriate parts ceiling for each shop he is responsible for. He can adjust this level to react to changing situations. 

     

 

REPORTING  - MERC+ has a reporting tool which will allow flexibility in creating reports. These reports, which are in .xls format, should be available on screen within minutes.

PAYMENT OF INVOICES BY MERC

Once manager approves work order, it is transmitted to RRIS for payment. Each vendor will have payment terms programmed into RRIS. Vendors will have access (via internet)to status of their w/o's. Vendors will soon know - once w/o has been approved, payment is automatic.

RRIS, the Maersk accounting system, will pay all Maersk STS & Non-STS w/o's.

Setup of RRIS for vendor will be handled through corporate. Local manager will provide required infomation.

Non-MERC payments will NOT be included with MERC payments - this to avoid confusion when vendor is posting payments

Methods of payment -  Paper check or Wire Transfer

Notifications - With each payment, MERC will send electronic file of reference numbers & amounts.


How does MERC know how to handle payments ? - Shop number indicates who to pay and mode indicates who to charge and what account to use. Codes will be used for statistical purposes.
