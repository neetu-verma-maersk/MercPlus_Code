NUMBERED SPARE PARTS

  • Parts used in repairs that are reported to MSL by inserting the part number.
  • Numbered parts are usually reefer or genset parts
  • By inputting the part numbers, the vendor is reimbursed for the part
    • at the rate mentioned in the master parts table
    • minus the manufacturers discount ( if applicable see SHOP TYPES)
    • plus the markup
  • Part number must be on the MASTER PART TABLE if they are to be successfully entered in MERC+.
    • If a part number, suspected to be correct, cannot be entered in MERC+, contact your area EMR manager.
  • Part numbers may be on the mode/code - part table and if so parts entered for that mode/code combination must appear on that table.
  • Part numbers should be entered as they appear on the box. They should include dashes ( - ) where necessary.
  • Vendor does not have the option of entering the cost of the part (instead of the part number).
  • Vendor can enter the cost shop supplies (grease, screws etc.) as shop supplied materials as long as he does not exceed:
    • The local shop contract
    • The mode/code material ceiling
       

 

 

SPARE PARTS - MASTER PARTS TABLE

Download the latest Master Parts Table

  • The MASTER PARTS TABLE is a list of all reefer and genset parts which are used for MSL and leased reefers and gensets. It also includes the manufacturer's list price for the parts.
  • Parts can be entered in the MASTER PARTS TABLE in two ways:
    1. Individually
    2. Via automated 'dump' the details of which are TBA.
  • When the vendor enters a valid part number for an STS code, MERC+ will supply the price of the part to the estimate. The pricing will take into account:
    • Manufacturer's discount (if applicable - depending on the SHOP TYPE).
    • Vendor's markup for freight and handling.
    • Import tax* - if applicable.
    • VAT or sales tax* - if applicable.
      * - taxes will be totaled for all shop supplied parts and shown on separate lines.
  • The part numbers are in the same format as is printed "on the box" so if mechancs write down the part number they use - it should be in the MASTER PARTS TABLE.
  • Obsolete parts are in the table and when entered, will "point" to the superceding part and give the price for that part (without changing the part number as entered). Thus vendors can use superceded parts without problem.
  • Part number which are entered incorrectly or part numbers which are not in the MASTER PARTS TABLE will not be accepted. If your vendor has a part which will not take - contact GLBEMR.
  • An STS code may have parts associated with it. see ASSOCIATE PARTS WITH MODE/CODE.
  • Fields in the MASTER PARTS TABLE
    • Parts Group
    • Part Designation fields - These 3 fields are "free text" and can be used by GLBEMR to mark specific parts or groups of part for reporting purposes etc.
    • Manufacturer's Code
    • Part Number
    • Quantity - How many pieces are in a part (i.e. dryers might come 6 to a box but used individually). This allows MERC+ to divide price by the pieces to arrive at the cost of the individual piece.
    • Description
    • Active Y/N
    • Core Part Required
    • Core Value - The value of the core part as assigned by manufacturer of the part.
    • Deduct Core Value - If the price of the core part is included in the list price - this switch is set to Y.
    • Comments
  • Format for the master parts table

MERC+ will hold 2 indicators on master parts table. One will determine if part is a core part or not, the other will determine if it is a Maersk part for shops 1 & 3.
The Estimate entry will display a message Warning of the core part registration - the Estimate review for conversion to WO will have a place to enter the core part serial number if a core part has been entered on Estimate. 

 

SUPERCEDED PARTS
If a part number has been superceded, MERC+ will still allow entry of that part number. MERC+ will take that part number and search for the superceding part number. It will then price the superceded part with the superceding part price. This process can/will work over multiple layers of superceded parts. In the example below, part ABC-2 supercedes part ABC-1. Entering ABC-1 on an estimate will yield a price of 1.10.

PART NUMBER DESCRIPTION PRICE REMARKS
ABC-1 ABC-2 0.00 SUPERCEDED PART
ABC-2 SCREW 1.10 LATEST PART

What makes Superceded Part feature work

  • The superceded part number and the superceding part number have to be on the master parts table both in the correct format.
  • The superceded part has to have the superceding part number in the description field in the correct format.
  • The superceded part has to have 0.00 for a price.

Initial load of part numbers to master part table will be done during development. At present there is no facility to automatically update the master parts table in 2004. Also there is no facility to update the mode/code-parts association table.
 

 

 

 

ERROR HANDLING WITH SPARE PARTS
If a part is not formatted correctly for MERC+ to know it is a superceded part, it will assume it is not, and look for that part number on the code/part table. Not being there will result in an error to the user that the part number is invalid.
If it is formatted correctly, but the new part number is not in the Code/part table, it will also result in an error to user.
 

SPARE PART CONTROL
Spare parts will be controlled in MERC+ in various ways, by switches and settings in fields of various tables.

  • At the part level –

    • Master Parts table – Mfg’s list of all (reefer and genset) parts

  • At the shop level - see SHOP TYPES

  • At the code level – for each code there will be:

    • Switch (Y/N) for mandatory Maersk supplied parts

      • These are parts which Maersk can obtain cheaply and so do not allow vendor to bill for these parts.

      • Default of switch will be N

      • If switch is set to Y – entry of that part no. is mandatory

      • For all shop types - enter part number and

        • Shop is not paid

        • Part number is reported to RKRP

    • Shop supplied Material Amount

      • This amount field will regulate amt of shop supplied material allowed

        • 0 = no shop supplied material allowed

        • 99999 = unlimited (default)

        • Anywhere in between

          • Estimates with amounts over the amount will NOT be allowed into MERC+ - a warning will be displayed.

  • At the Mode/Code-Parts tables – tables which associate various parts with a mode/code combination

    • Not all codes will have part numbers associated with them in code/parts table – if not; part numbers will be validated and priced from the master parts table.

    • If STS code does have an entry in the code/parts table it will be treated as follows:

      • Part entered will be checked against master parts table to look for superceding part.

        • If no superceding part – part number is checked against Mode/Code-Parts table

        • If is a superceding part – superceding part number is checked against Mode/Code-Parts table

      • ALL part numbers entered under that code must be in the code/parts table.

CORE PART

There are three fields in the MASTER PARTS TABLE which deal with cores.

  1. CORE PART REQUIRED - this tell MERC+ that the serial number from the removed core part must be captured.
    • MERC+ warns the that this number must be captured at the estimate submission stage.
    • MERC+ will not allow the user to convert the approved estimate to a work order without the number.
      • PROCEDURE IF CORE PART IS NOT AVAILABLE    
  2. CORE VALUE - Some spare parts have a core value - that is the hull of the defective part can be returned for possible core credit. This is controlled by the CORE Y/N switch at the part level.
  3. DEDUCT CORE VALUE - Some parts have their core value included in the price of the new part. This core refund must be deducted from the price of the part when displaying on the estimate.
    • This value is in USD and converted to local currency for pricing.

 

SHOP SUPPLIED MATERIALS


Non-Reefer/Genset spare part control in MERC+
 CONTRACTUAL AGREEMENTS GOVERN WHAT WILL BE PAID FOR NON-REEFER/GENSET PARTS.

  •  IF THERE IS A CONTRACT IN PLACE
    • Indicated by field in shop setup screen
    • It is hoped that contractual arrangements can be in .xls or other fixed format for importing in to MERC+ to avoid re-keying.
    • There are no part numbers for non-reefer/genset parts WITH THE EXCEPTION OF CHASSIS.
    • Contracts exist between CPH and local MSL at the country level. These contracts will list about 4 modes each with about 100 STS codes with the agreed for materials/parts in USD or local currency. This is what CPH will pay to local MSL when these repairs are performed
    • Contracts will exist between individual shops or vendors and local MSL listing the same 4 modes each with about 100 STS codes with the agreed for materials/parts in local currency. This is what vendor will be paid by local MSL when these repairs are performed.
      • If vendor inputs amount in shop materials field for an STS code at or under the contract rate – it will be accepted. If it is over the contract rate the system will give an error msg with the proper amount.
      •  STS codes which are not covered by the contract(s) are covered by the SHOP MATERIAL CEILING at the code level.
  • IF THERE IS NO CONTRACT IN PLACE BETWEEN VENDOR AND LOCAL MSL, MATERIALS CEILINGS FOR INDIVIDUAL CODES WILL APPLY.
    • It is EXTREMELY IMPORTANT that MSL personnel understand the global ceiling is a "safety net" only. It is NOT the standard amount to be charged. Personnel should not mention the global ceiling to vendor - they should constantly stress that the actual cost should be entered.

HOW SHOP IS PAID FOR NON-REEFER/GENSET PARTS

  • When an STS code is entered, system will check the shop to see if contract is in place between the vendor and local MSL.
    • When the approved estimate has been changed to a work order:
      • vendor will be paid at the contractual rate for those parts.
      • Local MSL will be paid, by CPH, for the parts at the contractual rate between CPH and local MSL
    • If there is not a vendor-local MSL contract – shop must enter amount which is checked against ceilings for individual STS codes.
